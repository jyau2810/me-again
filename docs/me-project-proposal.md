# 《我》游戏立项文档

版本：v0.1  
日期：2026-04-26  
文档用途：面向个人开发与未来合作者，用于确认第一阶段 Demo 范围、制作路线、资源需求、风险和验收标准。  
相关文档：[《我》创作蓝本](./me-creative-blueprint.md)、[《我》平台调研报告](./platform-research.md)

## 1. 项目概述

《我》是一款竖屏点触解谜冒险游戏。玩家扮演一个生活稳定但情感迟钝的成年男性，在现实生活与童年里世界之间往返：现实中的他疲惫、规整、很少主动取悦自己；进入里世界后，他变成小孩形象，用童年幻想理解学校、树影、车灯、漫画、柜子和密林。

游戏不是讲“逃离现实”，而是讲“把感受世界的能力带回现实”。主角最后仍回到平平淡淡的生活，但他重新能看见树、花、路灯和自己，也重新学会用一点点小事照顾自己。

## 2. 产品定位

| 项目 | 方案 |
| --- | --- |
| 游戏名 | 《我》 |
| 类型 | 竖屏点触解谜冒险 |
| 核心体验 | 成人现实与童年里世界交错，轻量探索、环境谜题、隐藏道具、情绪回收 |
| 平台优先级 | 微信小游戏优先，抖音小游戏第二，海外 Web/itch.io 后续验证 |
| 第一阶段目标 | 10-20 分钟竖切 Demo |

| 战斗与压力 | 无战斗、无死亡惩罚、低压力 |
| 叙事气质 | 温柔治愈、半自传虚构、结局明亮但里世界真相开放 |
| 团队假设 | 完全个人开发 + AI 辅助美术/音频 |

## 3. 目标用户

### 核心用户

- 喜欢叙事解谜、轻量探索、情绪向独立游戏的玩家。
- 对童年幻想、校园记忆、成长后情感钝化有共鸣的成年人。
- 能接受低压力、慢节奏、无战斗体验的小游戏或独立游戏用户。

### 平台用户假设

- 微信小游戏用户：更适合短会话、低门槛、无需安装的体验，因此 Demo 要在 60 秒内出现可点击、可变化、可回收的小谜题。
- 抖音小游戏用户：更依赖视觉钩子和短视频传播，因此后续宣传可突出“现实成人 / 里世界小孩”的形象变化。
- itch.io/Web 用户：更能接受作者表达和开发日志，适合作为海外低手续试玩渠道。

## 4. 核心体验与设计支柱

### 体验目标

玩家完成 Demo 后，应能记住三件事：

1. **同样的动作会让世界变化。** 绕教学楼不是移动行为，而是进入里世界的仪式。
2. **童年幻想可以变成解谜规则。** 粉笔箭头、课桌涂鸦、跑圈次数和隐藏道具共同决定谜题解法。
3. **主角看世界的方式变了。** 开局点击物件只显示“树”“小花”；结尾复访时，文本变得更生动，像主角重新愿意和世界发生关系。

### 核心循环

现实日常片段 → 发现入口动作 → 重复动作让场景逐步靠向里世界 → 小孩形象进入探索 → 解环境谜题或找到隐藏童年道具 → 回到现实并出现细微变化。

### 必须保留的特色系统

- **感知文本系统**：同一物件在不同阶段显示不同点击文本，表现主角从“只命名事物”到“重新感受事物”的变化。
- **入界动作系统**：重复同一动作让现实逐步异化，例如绕教学楼三到四圈。
- **隐藏童年道具**：道具不强调数值功能，重点是“没用但很酷”，例如半截粉笔、贴纸星星、橡皮碎屑。
- **儿童文字痕迹**：小孩留下的纸条短、直白、具体，用自造规则提示玩家。

## 5. 第一阶段范围：竖切 Demo

### Demo 目标

第一阶段只做 10-20 分钟竖切 Demo，用来验证核心循环、美术气质、触控操作、Cocos 小游戏构建路径和早期用户反馈。Demo 不承担完整故事表达，只证明《我》这套玩法和情绪是可成立的。

### 选择章节

Demo 选择第二章“绕完这一圈”，原因：

- 入界动作明确：围绕教学楼或操场重复绕行。
- 玩法特征独特：同样动作随着次数增加，场景逐渐靠向里世界。
- 场景资产可控：学校门口、操场、走廊、课桌即可支撑体验。
- 情绪回收清楚：成年主角重新发现自己曾经会玩。

### Demo 内容清单

| 模块 | 内容 |
| --- | --- |
| 现实开场 | 成年主角下班路过学校，玩家可点击树、小花、校门、路灯等物件，早期文本简短客观 |
| 入界动作 | 玩家围绕教学楼或操场移动三到四圈，每圈触发轻微场景变化 |
| 里世界形态 | 主角变成小孩形象，场景色彩变暖，出现粉笔箭头、课桌涂鸦和不合理空间 |
| 子场景 1 | 操场环线：跑道线变成迷宫线，玩家按跑圈规则打开下一段 |
| 子场景 2 | 粉笔走廊：墙上出现儿童涂鸦和箭头，提示真正的路线 |
| 子场景 3 | 旧课桌岛：课桌排列成岛屿，抽屉里藏有纸条和隐藏道具 |
| 主谜题 | 根据跑圈次数、课桌涂鸦和粉笔箭头，找到“真正的第四圈” |
| 隐藏道具 | 半截粉笔、贴纸星星 |
| 结尾回声 | 主角回到现实，桌面放下粉笔，复访物件文本变得更生动 |

### Demo 不做的内容

- 不做内购、广告、排行榜、复杂账号、多人玩法。
- 不做完整五章剧情。
- 不接入高复杂度云存档。
- 不做真实 IP 漫画角色或真实标题。
- 不做战斗、死亡惩罚或追逐恐怖段落。

## 6. 完整版本规划

完整版目标体量为 3-5 小时，共 5 个大章。每章包含现实段、入界动作、2-3 个里世界子场景、1 个主谜题、2-4 个隐藏碎片和 1 个现实回声。

| 章节 | 核心主题 | 入界动作 | 主玩法 |
| --- | --- | --- | --- |
| 灰色早晨 | 我已经很久没有认真看世界了 | 观察车窗与车灯 | 分辨车灯敌友、帮助友军通过夜路 |
| 绕完这一圈 | 我曾经能把学校跑成一扇门 | 绕教学楼或操场 | 跑圈异化、粉笔箭头、课桌涂鸦 |
| 纸片朋友 | 我曾经相信故事里的人会来陪我 | 翻动旧漫画 | 分镜重排、影射式原创纸片伙伴 |
| 柜子里的呼吸 | 我曾经会为自己发明安全的地方 | 开合柜门、躲猫猫 | 声音摆放、雨声、柜子空间 |
| 密林之后 | 我不需要知道答案，也可以重新出发 | 穿过树影和密林 | 手指小人合作、树洞与山腹秘境 |

完整版可在 Demo 验证后拆成阶段开发。每一章都必须能独立形成“入界动作 + 场景谜题 + 现实回声”的小闭环。

## 7. 技术方案

### 引擎路线

第一阶段采用 **Cocos Creator 小游戏优先**。立项建议按 Cocos Creator 3.x / 3.8 系列评估，实际版本以开工时的稳定版为准。

选择理由：

- 适合 2D 竖屏点触解谜。
- 可构建微信小游戏和抖音小游戏工程。
- 对个人开发者相对轻量，方便控制包体和资源。
- 后续可视情况导出 Web 版本，用于 itch.io 或其他海外 Web 测试。

### 需要注意

Cocos Creator 可以生成微信/抖音小游戏工程，但不是自动完成上线：

- 微信小游戏仍需微信开发者工具、AppID、平台配置、预览调试、审核与备案。
- 抖音小游戏仍需抖音开发者工具、AppID、平台配置、预览调试、审核与备案。
- 分享、广告、云存档、登录等平台 SDK 需要分别接入与测试。
- 第一阶段先不接复杂平台能力，只验证单机流程、基础存档和构建链路。

### 技术模块

| 模块 | Demo 需求 |
| --- | --- |
| 场景管理 | 现实学校、操场环线、粉笔走廊、旧课桌岛、结尾现实复访 |
| 点击交互 | 点击移动、点击调查、物件文本弹泡 |
| 感知文本 | 根据进度切换物件描述：开局现实、章节后、结尾复访 |
| 入界状态 | 记录绕圈次数，逐步改变场景表现 |
| 谜题状态 | 记录粉笔箭头、课桌涂鸦、第四圈条件 |
| 道具收集 | 半截粉笔、贴纸星星，进入收集册 |
| 存档 | 本地保存 Demo 进度、收集、文本状态 |
| 音频 | 背景音乐、脚步、粉笔、环境声、回归现实音效 |

## 8. 美术、UI 与音频方案

### 美术方向

主方向为“温暖现实 + 手绘幻想”。现实世界低饱和、普通可信；里世界更饱和，加入粉笔、纸片、贴纸、旧课桌、树影等童年质感。

Demo 所需美术资源：

- 成年主角与小孩主角基础立绘或小人动画。
- 现实学校门口、操场、走廊、课桌区域。
- 里世界变体：粉笔箭头、涂鸦、发光窗户、变形跑道线。
- UI：点击文本气泡、道具栏、收集册、章节标题。
- 隐藏道具：半截粉笔、贴纸星星。

### AI 辅助美术/音频

团队假设为完全个人开发，必要时使用 AI 辅助生成美术和音频：

- 概念图：用于 moodboard、场景氛围和角色方向。
- 2D 资产：可用 AI 生成草稿，再统一描线、调色和尺寸。
- UI 草案：可用 AI 生成贴纸、纸条、作业本风格参考，再手动整理为可用控件。
- 音效：粉笔、翻书、脚步、树叶、纸张摩擦等可用 AI 或可商用音效库生成/筛选。
- 音乐：先用短循环音乐验证氛围，后续根据反馈再决定是否定制。

所有 AI 生成或外部素材都要检查授权、平台风险和风格一致性。真实 IP、知名角色、可识别商标和版权音乐不能进入可发布版本。

## 9. 商业化与平台策略

### 第一阶段

Demo 免费发布或小范围测试，不做内购、不做排行榜、不做复杂账号，不把广告作为首要目标。目标是验证玩法、完成度、审核路径和玩家反馈。

### 平台顺序

1. **本地/Cocos 预览**：验证核心循环和触控。
2. **微信小游戏测试版**：验证构建、包体、基础审核路径。
3. **抖音小游戏预研**：在微信路径跑通后再适配。
4. **Web/itch.io Demo**：适合作为海外和独立游戏社区反馈渠道。

### 商业化后置

广告、买断、内购和大陆正式商业化都放到 Demo 验证之后。若进入大陆正式商业化，需要重新评估软著、备案、版号、出版单位、运营主体、防沉迷和支付能力。

## 10. 8-12 周制作排期

### 第 1-2 周：项目准备与原型

- 搭建 Cocos Creator 工程。
- 建立竖屏分辨率、场景切换、点击移动、文本气泡基础框架。
- 完成学校现实场景灰盒。
- 完成绕圈次数记录和简单场景变化。

验收：玩家能在竖屏场景中点击移动，绕圈次数能被记录，第三圈后场景出现可见变化。

### 第 3-4 周：核心循环

- 完成现实开场到入界的流程。
- 完成成年主角与小孩主角切换。
- 完成操场环线和粉笔走廊基础交互。
- 完成主谜题的第一版规则。

验收：从现实开场到进入里世界再到触发主谜题，可以连续游玩不中断。

### 第 5-6 周：内容填充

- 完成旧课桌岛。
- 加入半截粉笔和贴纸星星两个隐藏道具。
- 加入儿童纸条、课桌涂鸦和感知文本三档文本。
- 完成结尾现实回声。

验收：Demo 能从开头完整玩到结尾，包含主谜题、隐藏道具和结尾文本变化。

### 第 7-8 周：视听与体验打磨

- 替换关键临时资源，统一现实与里世界色彩。
- 加入基础音乐、脚步、粉笔、环境声。
- 调整提示、点击热区、移动速度和谜题节奏。
- 做一次完整自测并记录卡点。

验收：首次游玩者不看说明也能在 20 分钟内完成 Demo，过程中能理解“绕圈让世界变化”。

### 第 9-10 周：小游戏构建与平台准备

- 构建微信小游戏工程。
- 用微信开发者工具预览和调试。
- 检查包体、加载速度、分辨率适配和本地存档。
- 准备 Demo 简介、截图、隐私说明草案。

验收：微信开发者工具中可以完整跑通 Demo，无明显适配问题。

### 第 11-12 周：封版与小范围测试

- 修复测试反馈中的高优先级问题。
- 确认无真实 IP、版权音乐、未授权素材。
- 整理测试问卷或反馈表。
- 输出 Demo 版本说明和下一阶段建议。

验收：形成一个可给 5-10 名测试者体验的竖切 Demo 包，并收集可用于决定下一阶段的反馈。

## 11. 成本与资源需求

### 个人开发投入

- 每周稳定投入时间是最大成本。建议每周至少固定 10-15 小时，否则 8-12 周周期会明显拉长。
- 需要同时承担策划、程序、美术整合、音频整合、测试和平台配置。

### 可选成本项

- Cocos Creator 学习与插件成本。
- AI 图片/音频生成工具订阅。
- 可商用字体、音效库、素材包。
- 平台开发者账号或认证费用。
- 后续若进入正式商业化，可能产生软著、备案、出版/版号、外包和测试成本。

### 资源管理原则

- Demo 所有资源必须能复用到完整版第二章。
- 美术资源优先做少量高复用场景，不做一次性大场面。
- 音频优先覆盖关键反馈：点击、脚步、粉笔、入界、回归现实。
- 文本优先保证精确和有童心，数量少于质量。

## 12. 风险清单

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 个人开发范围过大 | Demo 延期或半成品化 | 第一阶段只做第二章竖切，完整五章不进入 Demo 范围 |
| Cocos 小游戏构建不熟 | 平台测试延后 | 第 1-2 周即建立最小构建验证，不等内容做完 |
| 谜题太隐晦 | 玩家卡住 | 用光线、粉笔线、儿童纸条和角色视线做渐进提示 |
| 文本太成人或太说教 | 情绪失真 | 遵守儿童文字指南，避免抽象总结 |
| AI 资源风格不统一 | 画面像拼贴 | 统一描线、调色、尺寸和材质，不直接混用多套风格 |
| IP 影射过近 | 审核或法律风险 | 蓝色胖猫等只作为影射式原创，不使用真实标题、标志道具和高相似组合 |
| 平台合规超预期 | 上线延期 | Demo 先免费验证，不做内购、广告、复杂账号 |
| 触控体验不顺 | 小游戏用户流失 | 热区足够大，交互反馈明确，少用精细拖拽 |

## 13. 验收标准

### Demo 内容验收

- 可完整游玩 10-20 分钟。
- 包含现实开场、入界动作、三个里世界子场景、主谜题、两个隐藏道具、结尾现实回声。
- 结尾复访同一场景时，至少 6 个物件使用感知文本变化。
- 至少 3 条儿童文字痕迹参与提示或情绪表达。

### 技术验收

- Cocos Creator 工程可在编辑器中完整运行。
- 可构建微信小游戏工程，并能在微信开发者工具中预览。
- 支持本地存档：进度、收集道具、感知文本阶段。
- 竖屏适配主流手机比例，关键按钮和点击热区不遮挡。

### 体验验收

- 首次玩家能在不看外部说明的情况下理解“绕圈会让世界变化”。
- 玩家能通过环境提示解出主谜题，不依赖开发者口头提示。
- 玩家能感知成年主角和小孩主角的状态差异。
- 结尾能通过物件文本变化传达主角视角变化，而不是靠大段解释。

### 合规与素材验收

- 不包含真实漫画、动画、游戏 IP 标题或角色形象。
- 不包含未经授权的字体、音乐、音效或素材。
- 不接入非必要权限。
- 不做内购、排行榜、复杂账号或强联网功能。

## 14. 下一步

立项文档确认后，建议进入 Demo 制作准备：

1. 建立 Cocos Creator 工程与版本管理。
2. 输出第二章 Demo 的详细关卡流程图。
3. 制作现实学校与里世界的 moodboard。
4. 写第一版谜题脚本和感知文本表。
5. 用灰盒完成从现实到入界再到回归的最小可玩流程。
