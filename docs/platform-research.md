# 《我》平台调研报告

调研日期：2026-04-26  
适用对象：个人独立开发者，优先考虑中国大陆轻量移动端小游戏，兼顾海外低手续发布。  
重要说明：本文是产品与发行决策参考，不构成法律意见。大陆平台、版号、备案、防沉迷、广告和支付规则变化频繁，正式提交前必须以平台后台、主管部门最新要求和专业合规意见为准。

## 1. 结论

《我》适合先按轻量竖屏点触解谜游戏推进，发行路径建议分三层：

1. **大陆验证优先：微信小游戏免费原型。** 先做 10-20 分钟竖切版，不开内购，不急于广告变现，用来验证题材、操作、审核路径和用户反馈。
2. **大陆扩展备选：抖音小游戏、TapTap 测试/试玩。** 抖音流量强但资质链路更重；TapTap 更适合面向核心玩家展示、预约或测试，但正式下载/付费/内购会明显进入版号、防沉迷和备案链路。
3. **海外低手续验证：itch.io 与 CrazyGames。** Web 版或 HTML5 版本适合低成本测试口碑；后续再根据完成度考虑 Google Play、App Store 或 Steam。

当前最稳的第一阶段不是“正式商业化”，而是“免费、轻量、可审、可测”。广告可作为第二阶段商业化选项，但不应被视为绕开版号或资质要求的捷径。

## 2. 大陆合规要点

### APP/小程序备案

工信部 2023 年发布的 APP 备案通知明确将“小程序、快应用等分发”纳入移动互联网应用程序备案范围；在中国大陆境内从事互联网信息服务的 APP 主办者应履行备案手续，未备案不得从事 APP 互联网信息服务。小程序备案通常通过微信、抖音等平台后台提交，涉及平台初审、工信部短信核验和通管局审核。

对《我》的影响：

- 微信/抖音小游戏正式发布前，要按平台要求完成小程序或小游戏备案。
- 若涉及联网服务、账号、云存档、数据收集或广告，备案、隐私合规和数据合规压力会增加。
- 第一阶段应尽量减少联网能力，只保留必要的存档和统计。

### 游戏版号/ISBN

国家新闻出版署持续公示国产网络游戏审批信息，公示表包含游戏名称、出版单位、运营单位、批复文号、出版物号和批准时间。2026 年 3 月份国产网络游戏审批信息已经在国家新闻出版署结果公示中列出，说明版号审批仍是国内网络游戏正式运营的重要门槛。

对《我》的影响：

- 如果在大陆地区做正式商业化运营，特别是付费下载、内购、开放下载或平台判定的商业运营，通常应提前评估版号需求。
- 个人开发者直接办理版号难度高，通常需要出版单位和运营主体配合。
- 早期应避免把大陆付费下载和内购作为第一目标。

### 软件著作权

《计算机软件著作权登记办法》规定，申请软件著作权登记需提交登记申请表、软件鉴别材料和相关证明文件，中国版权保护中心是软件登记机构。小游戏平台常会要求软著与游戏名称、主体信息一致，尤其在抖音、TapTap 等平台资质审核中体现明显。

对《我》的影响：

- 若计划在大陆平台长期运营，建议早期规划软著名称，例如“我游戏软件 V1.0”或平台可接受的等价名称。
- 名称《我》很有作者性，但平台可能要求名称具备识别度、避免过短或通用词争议。建议保留内部主标题《我》，提交平台时准备备选备案名，例如《我：密林之后》或《我游戏软件》。

### 防沉迷、实名与隐私

大陆游戏平台通常会要求未成年人防沉迷和实名认证。TapTap 文档明确将防沉迷认证、隐私合规列入多种上线或试玩场景。若游戏包含联网、下载、付费或面向未成年人，合规要求会更重。

对《我》的影响：

- 题材温和，但并不自动豁免防沉迷与实名要求。
- 如果先做微信/抖音小游戏，应优先使用平台已有能力，减少自建账号系统。
- 隐私策略应保持极简：不采集通讯录、定位、人脸、设备敏感权限；统计只采集必要的匿名行为数据。

## 4. 大陆平台分析

### 微信小游戏

优势：

- 用户触达方便，适合轻量竖屏点触和章节式游玩。
- 不要求玩家安装独立 App，适合情绪向短体验传播。
- 微信生态适合“分享给朋友看一个小而私人的游戏”。

风险：

- 游戏类目、备案、资质、支付、广告能力都需要在微信公众平台后台核验。
- 个人主体的虚拟支付能力通常受限；早期不应设计内购。
- 如果游戏名只叫《我》，可能在搜索、备案、审核中被认为识别度不足，需要提前准备平台展示名。

建议：

- 第一阶段做免费原型：不开内购，不做强联网，不做排行榜，不要求登录。
- 技术上优先保证竖屏点触、低包体、快速进入、短会话。
- 商业化暂定为后置：先验证内容，再评估流量主广告、企业主体或版号路径。

### 抖音小游戏

优势：

- 短视频分发强，适合用“成年主角进入童年里世界”的视觉变化做传播点。
- 竖屏小游戏天然适合抖音用户习惯。

风险：

- 抖音小游戏资质规范明确强调软著真实性、唯一性、主体一致性；版号要求游戏名称、运营单位、出版物号等与材料一致。
- 基础信息审核对名称、简介、图标、详细介绍都有较细规则，禁止侵权、引流、测试信息、夸张商业话术等。
- 轻量叙事游戏需要在 10-60 字简介和 120-300 字详细介绍中准确表达玩法，不能写成纯文学介绍。

建议：

- 抖音作为第二平台，等微信或 Web 原型验证后再进入。
- 先准备资质一致的正式名称、软著名称和平台简介。
- 宣传短视频可以突出“现实成人/里世界小孩”的表情变化；影射式漫画伙伴只放在游戏内容里，不使用真实 IP 名称、角色形象或容易让用户误认授权关系的宣传暗示。

### TapTap

优势：

- 用户更能接受独立游戏、叙事解谜和试玩版。
- 开发者文档对不同上线方式、资质和防沉迷要求描述较清楚。
- 适合积累愿望单、预约、测试反馈和开发日志。

风险：

- TapTap 文档明确指出正式上线、开放下载或内购通常需要官方游戏出版物号；无版号时更适合预约、信息展示或开放试玩。
- 联网、内购、包体下载等场景会叠加 ICP 备案、防沉迷、隐私合规等要求。

建议：

- 不作为第一个移动小游戏平台，但可以作为独立游戏社区阵地。
- 在有 10-20 分钟可玩 demo 后开页面，做开发日志和测试招募。
- PC 完整版若推进，可同步考虑 Steam 与 TapTap PC。

## 5. 海外平台分析

### itch.io

优势：

- 手续轻，适合独立开发者快速发布免费 demo、Web 版或 PC 包体。
- 创作者可设置最低价格为 0，允许玩家自愿付费。
- 页面自定义能力强，适合呈现《我》的作者性和创作蓝本。

风险：

- 平台流量不自动保证，需要外部传播。
- 若使用付费内容、成人内容或第三方素材，仍要遵守平台、支付和素材授权规则。

建议：

- 第一海外目标。发布英文/中文双语页面，先做 free demo。
- 用 devlog 记录“半自传虚构、童年幻想、现实与里世界交错”的创作过程。

### CrazyGames

优势：

- 面向 Web 游戏，适合 HTML5、轻量小游戏和移动/桌面浏览器。
- 平台提供广告分成、统计、云存档等能力。

风险：

- 会经过 QA 和质量审核；叙事解谜需要有清晰可玩性，不能只是氛围展示。
- 需要适配横竖屏、加载速度和平台 SDK。

建议：

- 作为 Web 版第二海外平台。
- 将竖屏点触章节做成 10-15 分钟可完成的 demo，避免过长文本。

### Google Play

优势：

- Android 全球发行渠道成熟。
- 适合后续独立 App 版。

风险：

- 完整分发开发者账号有一次性 25 美元注册费。
- Google Play 对新个人开发者账号有 closed test 要求：至少 12 名测试者连续 14 天 opt-in 后才能申请生产访问。
- 隐私、数据安全、内容分级、广告 SDK 合规都需要单独处理。

建议：

- 不作为第一阶段目标。
- 若小游戏验证成功，再考虑封装 Android App。

### App Store

优势：

- iOS 用户付费习惯较好，适合高完成度情绪向独立游戏。
- TestFlight 适合小规模测试。

风险：

- Apple Developer Program 年费 99 美元。
- Apple 文档说明，中国大陆区游戏需要审批号，上传 ISBN 核发单或批复文件等材料。
- iOS 审核对隐私、数据、内购、第三方 SDK 都较严格。

建议：

- 海外 iOS 可以作为后期目标。
- 中国大陆区 iOS 不作为第一阶段目标，除非已经规划版号路径。

### Steam

优势：

- 更适合 3-5 小时完整版。
- 叙事解谜、个人表达、温柔治愈题材在 Steam 有相对清晰的独立游戏受众。

风险：

- 每个产品需支付 100 美元 Steam Direct 费用；产品达到至少 1000 美元调整后总收入后可抵回。
- 需要更完整的 PC 交互、手柄/键鼠适配、存档、成就、页面物料和客服准备。

建议：

- 作为完整版目标，而不是竖切原型目标。
- 若移动小游戏验证有效，可扩展为 PC 完整版《我》。

## 6. 商业化路径

### 第一阶段：免费原型

目标是验证核心体验，不追求收入。

- 平台：微信小游戏、itch.io Web demo。
- 内容：10-20 分钟，1 个现实段、1 个入界动作、1 个里世界、1 个主谜题、若干隐藏道具。
- 变现：无内购、无广告、无付费墙。

### 第二阶段：轻广告或赞助

目标是验证传播与留存。

- 平台：微信/抖音小游戏、CrazyGames。
- 前提：确认平台对广告变现、游戏资质、版号和主体的最新要求。
- 方式：激励视频不适合《我》的情绪体验；若使用广告，优先选择章节结束后的低打扰广告，或平台外的赞助/打赏。

### 第三阶段：完整版买断

目标是把《我》做成 3-5 小时完整独立游戏。

- 平台：Steam、itch.io、Google Play、App Store 海外区。
- 定价：PC 完整版可采用低价买断；移动端可免费 demo + 完整版买断或平台允许的付费方式。
- 大陆正式商业化：需重新评估版号、软著、出版单位、运营主体、备案、防沉迷和支付资质。

## 7. 素材与 UI 参考平台

当前创作蓝本阶段建议先做 moodboard，不直接把外部 UI 素材当最终资产。正式制作时再决定购买、使用免费素材或定制美术。

### 免费或低成本

- [Kenney](https://kenney.nl/assets)：大量 2D、3D、UI、音效资源；其支持页说明 asset pages 上的游戏资产为 CC0，可商用且不要求署名。
- [OpenGameArt](https://opengameart.org/content/faq)：可商用资源多，但许可证复杂。优先选择 CC0 或 OGA-BY；谨慎使用 CC-BY-SA、GPL 等可能影响 DRM、署名或衍生作品发布的许可证。
- [itch.io Game Assets](https://itch.io/game-assets)：免费和付费资源都有，风格丰富；必须逐项查看作者许可、AI 生成声明、署名要求和商用限制。

### 付费素材市场

- [Unity Asset Store](https://unity.com/legal/as-terms)：资源丰富，适合 Unity 项目；多数资产可用于商业项目，但需避开 restricted assets，并遵守 Unity Asset Store EULA。
- [GameDev Market](https://www.gamedevmarket.net/about/licences/)：付费 2D、GUI、音频资源较多；其 Pro Licence 允许在免费或商业游戏中使用授权资产，但不获得独占权。
- [Steam / Epic / Humble 等资产包](https://kenney.itch.io/kenney-game-assets)：可关注大型 bundle，但要保存购买记录和 license。

### 对《我》的素材策略

- UI：不要直接套常见奇幻 RPG UI。需要“普通现实 + 手绘幻想”的柔和界面，按钮像纸片、贴纸、课桌标签、旧作业本页签。
- 场景：若用素材包，应尽量只作为临时原型资源；最终场景最好定制，避免游戏显得像通用素材拼贴。
- 字体：中文字体必须确认商用授权。优先使用开源可商用字体，并保存 license。
- 音频：童年感可来自环境声、粉笔、风、翻书、车窗、脚步、树叶，不宜使用版权不明的童年动画音乐。

## 8. 推荐路线图

### 0-1 个月：创作与技术验证

- 完成创作蓝本和平台策略。
- 做 moodboard、角色形象草图、1 个现实场景和 1 个里世界概念图。
- 选定引擎：优先考虑适配小游戏的 Cocos Creator、LayaAir 或 Unity 小游戏导出方案。

### 1-3 个月：微信小游戏竖切

- 完成 10-20 分钟 demo。
- 不做内购、排行榜、复杂账号、社交关系链。
- 准备隐私政策、备案材料、基础审核文案。
- 若平台要求软著或自审材料，按后台最新要求准备。

### 3-6 个月：双平台测试

- 微信小游戏小范围测试。
- itch.io 发布 Web/PC demo。
- 根据反馈调整谜题密度、文本量、触控体验和情绪表达。

### 6 个月后：商业化决策

- 如果移动端数据好：考虑抖音小游戏和 CrazyGames。
- 如果叙事口碑好：扩展 PC 完整版，规划 Steam 页面。
- 如果要大陆正式商业化：启动软著、版号、主体、出版单位和防沉迷方案评估。

## 9. 关键风险

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 版号和资质路径不明确 | 大陆商业化延期 | 第一阶段只做免费原型，商业化后置 |
| 标题《我》过短或通用 | 平台审核/搜索识别风险 | 保留艺术标题，同时准备平台提交名 |
| 外部 IP 侵权 | 审核失败或法律风险 | 漫画伙伴采用影射式原创，例如蓝色胖猫、粗眉毛小孩；不使用真实标题、真实角色形象、标志道具或高相似组合，也不在宣传中暗示关联 |
| 现成素材风格不统一 | 破坏作者性 | 原型可用素材包，最终关键场景定制 |
| 叙事太重导致小游戏留存低 | 前几分钟流失 | 开场 60 秒内给出可点触、可变化、可回收的小谜题 |
| 隐私与 SDK 合规 | 审核失败 | 尽量少接 SDK，先不做账号系统 |
